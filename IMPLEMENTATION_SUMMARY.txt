FEATURE OVERVIEW
────────────────────────────────────────────────────────────────────────────────
Instead of BLOCKING access at login when subscription expires, the system now:

1. ✅ ALLOWS login with expired subscription
2. ⚠️  SHOWS warning toast on dashboard load
3. 🔄 PREVENTS action execution (like creating events)
4. 💳 POPS UP subscription renewal modal automatically
5. 🔄 ENABLES user to renew subscription in-context

────────────────────────────────────────────────────────────────────────────────
BEFORE vs AFTER
────────────────────────────────────────────────────────────────────────────────

BEFORE (OLD FLOW):
  User with expired sub tries to login
         ↓
  ❌ LOGIN BLOCKED - Error message shown
  ❌ Cannot access dashboard
  ❌ Frustrated user

AFTER (NEW FLOW):
  User with expired sub tries to login
         ↓
  ✅ LOGIN SUCCEEDS
         ↓
  Dashboard loads
         ↓
  ⚠️ Warning toast: "Subscription expired. Click 'Get Subscription' to renew."
         ↓
  User tries to create event / generate QR
         ↓
  ❌ API returns: {"error": "subscription_expired"}
         ↓
  🔴 Toast: "Your subscription has expired. Please renew to continue."
         ↓
  💳 POPUP MODAL OPENS AUTOMATICALLY
     - Shows ⏰ icon
     - Shows "Subscription Expired" heading
     - Shows 3 renewal plans
     - User selects plan → QR code appears → Payment flow
         ↓
  ✅ After payment, page auto-refreshes with renewed subscription
